### Install lldb
```
sudo apt install lldb
```

### Compile with debugging information
```
gcc -g3 program.c -o program
```

### Breakpoint in main
```
lldb ./program
(lldb) breakpoint set --name main
(lldb) run
```

### Step-by-step execution
```
next      # next line (does not enter functions)
step      # enters the function
finish    # exit function
continue  # continue
quit      # exit
```

### Inspect variables
```
frame variable # local variables
expr x         # prints the value of x
```
### Strategic breakpoints
```
breakpoint set --file program.c --line 42
```

### Conditions for finding logical errors
- Example: Suppose you expected i to never be negative. You can do:
```
breakpoint set --file program.c --line 60 --condition “i < 0”
```

# Basic use of Valgrind

| Purpose                         | Basic command                                                    | Description                                                                |
|-----------------------------------|--------------------------------------------------------------- ----|----------------------------------------------------------------------------|
| General memory analysis       | `valgrind ./program`                                             | Detects common memory errors (invalid reads/writes, leaks)  |
| Detect memory leaks         | `valgrind --leak-check=full ./program`                           | Reports memory leaks with detailed information                         |
| Show complete stack traces  | `valgrind --leak-check=full --show-leak-kinds=all ./program`     | Shows all types of leaks, including detailed stack traces     |
| Ignore errors from certain libraries   | `valgrind --leak-check=full --gen-suppressions=all ./program`    | Generates rules to ignore known errors (false positives)            |
| Save output to file          | `valgrind --log-file=valgrind.log ./program`                     | Saves Valgrind output to a file instead of the console          |
| View uninitialized memory       | `valgrind --track-origins=yes ./program`                         | Shows the origin of uninitialized memory used                      |
| Check memory usage by function | `valgrind --tool=massif ./program`                            | Analyzes dynamic memory usage over time with **Massif**    |
| View Massif output       | `ms_print massif.out.pid`                                         | Displays the report generated by Massif in readable format                  |
| Check race conditions | `valgrind --tool=helgrind ./program`                             | Detects synchronization errors and race conditions between threads     |
